## 1. 项目总览

### 1.1 这个项目做什么？

输入一条心电图（12 导联 ECG）+ 病人信息 → 输出诊断报告 + 推理链 + 标注波形图。

核心创新：不是传统的单模型分类，而是让 **LLM 驱动的 AI Agent 自主规划诊断步骤**，调用各种 ECG 分析工具，自我反思验证结果。

### 1.2 四层架构

```
┌─────────────────────────────────────────────┐
│         Phase 4: AI Agent 框架               │  ← 核心创新
│   LLM 规划 → 工具调用 → 自反思 → 诊断报告      │
├─────────────────────────────────────────────┤
│         Phase 3: 情境建模                     │
│   ECG Token + 文本编码 + 元数据 → 多模态融合    │
├─────────────────────────────────────────────┤
│         Phase 2: ECG 基础模型                 │
│   4个骨干网络 + 分类器 + 异常检测 + 训练器      │
├─────────────────────────────────────────────┤
│         Phase 1: 数据管道                     │
│   WFDB加载 → 滤波 → 归一化 → 增强 → Dataset   │
└─────────────────────────────────────────────┘
```

### 1.3 技术栈

| 组件 | 技术 |
|------|------|
| 深度学习框架 | PyTorch 2.x |
| ECG数据加载 | wfdb (PhysioNet官方库) |
| 信号处理 | scipy.signal (SOS滤波, resample_poly) |
| LLM接口 | OpenAI兼容 (DeepSeek / GPT / vLLM) |
| 前端Demo | Gradio (Phase 6) |

## 3. Phase 1: 数据管道

### 3.1 全景

Phase 1 负责将 PhysioNet 2020 Challenge 的原始 WFDB 格式数据转化为 PyTorch 可用的标准化数据。

**数据流**: `.hea + .mat` → `ECGSample` → `ECGPreprocessor` → `.npy` 文件 → `ECGDataset`

### 3.2 `loader.py` — 数据加载器

**核心类**: `ECGSample` (数据容器) + `ECGLoader` (批量加载器)

#### ECGSample

```python
sample = ECGSample(
    record_id="A0001",    # 记录标识
    signal=np.ndarray,    # 12导联ECG, shape (12, L), 单位mV
    fs=500.0,             # 采样率Hz
    metadata={"Age":"74","Sex":"Male","Dx":"59118001",...},
    source="cpsc_2018"    # 数据来源
)
```

**计算属性**:
- `duration` → 信号时长(秒)，`fs≤0`时安全返回`0.0`
- `age` → 解析年龄，NaN/inf/负数/>122岁 → `None`
- `sex` → 标准化为 "Male"/"Female"/"Unknown"
- `dx_codes` → SNOMED CT诊断码列表(只返回纯数字代码)

#### ECGLoader

```python
loader = ECGLoader("path/to/training")       # 自动发现6个数据源
loader.sources  # ['cpsc_2018','cpsc_2018_extra','georgia','ptb','ptb-xl','st_petersburg_incart']

# 单条加载
sample = loader.load_record("cpsc_2018/g1/A0001")

# 批量迭代(生成器,内存友好)
for sample in loader.iter_records(sources=["cpsc_2018"], max_records=100):
    process(sample)

# 快速统计
stats = loader.get_statistics(max_records=500)
```

**关键技术细节**:
1. `.mat`文件不是MATLAB格式! 是WFDB format 16 (16位差分编码, 12通道交错)
2. 使用`wfdb.rdrecord()`自动解码, 无需手动还原差分
3. `wfdb`会去掉`.hea`注释行的`# `前缀, 所以正则匹配用`^Age:`而非`^# Age:`
4. 正则模式预编译缓存(`_meta_patterns`), 43K条记录不重复编译

### 3.3 `preprocessor.py` — 信号预处理

**核心类**: `ECGPreprocessor` — 6步处理管道

```python
prep = ECGPreprocessor(
    target_fs=500,          # 目标采样率
    target_length=4096,     # 目标长度(~8.2秒)
    bandpass_low=0.5,       # 带通滤波低截止
    bandpass_high=45.0,     # 带通滤波高截止
    notch_freq=50.0,        # 陷波频率
    normalization="zscore", # 归一化方式: zscore/minmax/none
    outlier_threshold=5.0,  # 异常值裁剪阈值(σ)
)

# 一步处理
clean = prep(raw_signal, original_fs=500)

# 批量分段(用于长记录)
segments = prep.segment(long_signal, segment_length=2048, overlap=0.5)
```

**处理步骤**:
```
原始(12, L_orig) @ fs_orig
 → 带通滤波 0.5-45Hz (Butterworth 4阶, SOS形式, sosfiltfilt)
 → 陷波滤波 50Hz (fs过低时自动跳过)
 → 重采样至500Hz (scipy.signal.resample_poly, 多相滤波+抗混叠)
 → 分段至4096样本 (过长→居中裁剪, 过短→对称零填充)
 → 逐导联z-score归一化 (仅对信号区计算统计量, 不受零填充影响)
 → 异常值裁剪5σ (先copy再clip, 不修改原数组)
```

**关键设计**:
1. **SOS滤波**: 比传统ba形式数值更稳定 (二阶节级联)
2. **resample_poly**: 带抗混叠滤波, 保真ECG波形(QRS高频成分)
3. **填充感知归一化**: 统计量只计算非填充区域, 防止零填充稀释std
4. **Nyquist安全**: 带通上限≥0.99×Nyquist→抛异常, 陷波≥0.99×Nyquist→跳过

### 3.4 `augmentor.py` — 数据增强

**核心类**: `ECGAugmentor` — 6种ECG专用增强

```python
aug = ECGAugmentor(random_seed=42)
augmented = aug(ecg_signal, fs=500)            # 全部增强
augmented = aug(ecg_signal, apply_prob=0.5)    # 50%概率

# 可独立控制的6种增强:
# 1. baseline_wander  — 随机正弦基线漂移(freq 0.1-0.5Hz)
# 2. gaussian_noise   — 高斯噪声(σ 0.01-0.05×signal_std)
# 3. time_warp        — 时间扭曲(scale 0.8-1.2, 边界填充)
# 4. amplitude_scale  — 逐导联幅度缩放(0.8-1.2×)
# 5. lead_dropout     — 随机置零1-2导联(模拟电极脱落)
# 6. segment_shuffle  — 分段打乱(对比学习专用)
```

**关键设计**:
- 使用`np.random.Generator`(新版RNG, fork-safe)
- 支持单样本`(12,L)`和批量`(N,12,L)`模式
- 每种增强可独立开关(消融实验用)

### 3.5 `label_extractor.py` — 标签编码

**核心类**: `LabelExtractor` — 27类SNOMED CT ↔ 多标签编码

```python
le = LabelExtractor(num_classes=27)

# 编码: SNOMED CT → multi-hot向量
labels = le.encode(["426783006", "164889003"])         # → np.array([1,1,0,...])  (27,)
labels = le.encode(["426783006"], format="indices")     # → [0]
labels = le.encode(["426783006"], format="names")        # → ["Sinus Rhythm"]

# 解码: 概率向量 → [(snomed, name, prob), ...]
decoded = le.decode(prob_vector, threshold=0.5)

# 等价代码查找
name = le.get_class_name("39732003")  # → "Left Bundle Branch Block"
```

**27个类别**:
| 分类 | 数量 | 示例 |
|------|------|------|
| 节律(rhythm) | 9 | 窦性心律、房颤、房扑、窦缓、窦速、PAC、PVC、VT |
| 传导(conduction) | 8 | LBBB、RBBB、I°AVB、II°AVB、III°AVB、IRBBB、LAFB、WPW |
| 形态(morphology) | 10 | ST压低、ST抬高、T波倒置、心梗、心肌缺血、VEB、QT延长、RVH |

**42个SNOMED CT代码映射**: 27个规范代码 + 15个等价代码(如`39732003→164909002`)

### 3.6 `metadata_parser.py` — 元数据解析

```python
parser = MetadataParser()
parsed = parser.parse(metadata)
# → {"age": 0.617, "age_raw": 74, "sex": 1, "dx_codes": [...], ...}
```

- Age: 归一化`[0,1]`或`-1.0`(未知), 含NaN/inf检查
- Sex: `0=Female, 1=Male, 2=Unknown`
- 中英文症状/病史关键词匹配(chest_pain/胸痛等)

### 3.7 `dataset.py` — PyTorch Dataset

**3种Dataset + 1个DataModule**:

```python

ds = ECGDataset("data/physionet2020/processed", split="train", augment=True)
```

## 4. Phase 2: ECG 基础模型

### 4.1 全景

构建ECG深度学习模型体系: 经典信号处理特征 + 4个深度学习骨干 + 分类器 + 训练器。

### 4.2 特征提取 (`feature_extraction/`)

**经典信号处理方法(非DL)**, 给Agent提供可解释的ECG测量值。

```python
# R峰检测
detector = RPeakDetector(method="pan_tompkins")
result = detector.detect(ecg, fs=500)
# → {r_peaks, rr_intervals, heart_rate: 72.5, rhythm: "normal rate (regular)"}

# 多导联共识检测(30ms容差窗口)
result = detector.detect_multi_lead(ecg_12lead, fs=500)

# HRV分析
analyzer = HRVAnalyzer()
metrics = analyzer.analyze(rr_intervals)
# → {sdnn, rmssd, pnn50, lf_power, hf_power, lf_hf_ratio, sd1, sd2, sample_entropy, ...}

# QT分析
qt = QTAnalyzer().analyze(ecg, r_peaks, fs=500)
# → {qt_ms, qtc_bazett, qtc_fridericia, qrs_duration_ms, pr_interval_ms}

# 特征聚合(22维向量)
bank = FeatureBank()
features = bank.extract_all(ecg_12lead, fs=500)
vector = bank.to_vector(features)  # → np.array (22,)
```

**22维特征向量**:
`[heart_rate, hr_std, num_beats, mean_hr, sdnn, rmssd, pnn50, cvrr,
  lf_power, hf_power, lf_hf_ratio, total_power,
  sd1, sd2, sd_ratio, sample_entropy,
  qt_ms, qtc_bazett, qtc_fridericia, qtc_framingham,
  qrs_duration_ms, pr_interval_ms]`

### 4.3 骨干网络 (`backbone/`)

**统一接口** (所有骨干遵循):

```python
class ECGBackbone(nn.Module):
    def forward(x: Tensor) -> Tensor:
        # x: (B, 12, L) → output: (B, feature_dim)
        ...
    @property
    def feature_dim(self) -> int: ...
```

**4种骨干对比**:

| 模型 | 文件 | 参数量 | 特征维 | 风格 |
|------|------|--------|--------|------|
| ResNet1D-34 | `resnet1d.py` | 7.2M | 512 | 标准CNN基线 |
| xResNet1D-101 | `xresnet1d.py` | 7.6M | 512 | 挑战赛冠军方案 |
| InceptionTime | `inception_time.py` | 252K | 128 | 多尺度轻量 |
| ECG Transformer | `transformer_encoder.py` | 6.5M | 256 | 自注意力创新 |

**ECG Transformer 架构** (推荐创新骨干):
```
(B, 12, 4096)
  → Lead-wise Conv1D(kernel=49, stride=24): (B, 256, 170) per lead
  → Flatten: (B, 2040, 256)
  → Position Encoding (sinusoidal)
  → TransformerEncoder ×8: d_model=256, nhead=8, FFN=1024
  → Global mean pooling: (B, 256)
```

### 4.4 分类器 (`classifiers/`)

```python
# 心律失常分类器
model = ArrhythmiaClassifier(backbone, num_classes=27)
logits = model(ecg)              # → (B, 27)
probs = model.predict(ecg)       # → sigmoid(logits)

# 损失函数
asl = AsymmetricLoss(gamma_neg=4.0, gamma_pos=0.0, clip=0.05)
loss = asl(logits, targets)      # 处理极端类别不平衡

# 异常检测器(VAE)
detector = ECGAnomalyDetector(backbone, latent_dim=128)
recon, mu, logvar, z, features = detector(ecg)
total_loss = detector.loss_fn(...)
detector.fit_threshold(normal_loader)  # 在正常ECG上校准
score = detector.anomaly_score(ecg)    # → 重建误差
is_abnormal = detector.is_anomaly(ecg) # → True/False
```

**Asymmetric Loss 原理**:
- 对负类施加更强的聚焦(`γ_neg=4`), 防止简单负样本主导梯度
- 概率偏移(`clip=0.05`): `p_m = max(p - m, 0)`, 抑制简单负样本
- 从论文公式正确实现(减法在概率空间, 不是乘法在logit空间)

### 4.5 训练器 (`trainer.py`)

```python
trainer = ECGTrainer(model, device="cuda")

# 步骤1: SimCLR对比预训练
trainer.train_contrastive(
    contrastive_loader,
    epochs=100, temperature=0.07
)

# 步骤2: 多标签微调
history = trainer.train_multilabel(
    train_loader, val_loader,
    epochs=50,
    loss_fn=AsymmetricLoss()
)

# 步骤3: 评估
metrics = trainer.evaluate(test_loader)
# → {macro_auc, macro_f1, challenge_score, mAP}
```

**训练优化**:
- AdamW + CosineAnnealing + LinearWarmup
- 混合精度(AMP)自动启用
- 梯度裁剪(max_norm=1.0)
- 早停(patience=10)
- 最佳模型自动保存

## 5. Phase 3: 情境建模

### 5.1 全景

将 ECG 信号 + 临床文本 + 患者元数据 三模态融合为统一诊断模型。

### 5.2 三模态编码器

```python
# ECG → Embedding (VQ-VAE或SimpleECGProjector)
proj = SimpleECGProjector(input_channels=12, output_dim=256)
ecg_emb = proj(ecg)                    # (B, 256)

# 或VQ-VAE离散化
vq = ECGTokenizer(input_length=4096)
recon, tokens, vq_loss, perplexity = vq(ecg)  # tokens: (B, 512)

# 文本 → Embedding
te = TextEncoder(model_name="pubmedbert")     # BioBERT/PubMedBERT
text_emb = te(["Patient presents with chest pain..."])  # (B, 768)

# 元数据 → Embedding
me = MetadataEncoder(output_dim=256)
meta_emb = me(age=tensor, sex=tensor, source=list)  # (B, 256)
```

### 6.1 全景

## 8. 关键设计决策

### 8.1 为什么用 wfdb 而非 scipy 读 .mat？

PhysioNet 的 `.mat` 文件不是 MATLAB 格式，是 **WFDB format 16**（16位差分编码）。`scipy.io.loadmat()` 会报错。`wfdb.rdrecord()` 自动解码差分、识别导联名、解析元数据。

### 8.2 为什么用 SOS 滤波而非 ba 形式？

二阶节级联(SOS)比分子分母多项式(ba)数值更稳定。4阶 Butterworth 的 ba 系数幂次高，窄带/低频时可能产生数值误差。

### 8.3 为什么用 resample_poly 而非线性插值？

ECG 含 QRS 波群（~10ms 宽的尖锐脉冲，高频成分接近 Nyquist）。线性插值会模糊 QRS 形态。`resample_poly` 带抗混叠滤波，保真度高。

### 8.4 为什么归一化要排除零填充区域？

预处理中短信号会被零填充到 4096 样本。如果对整个填充后信号计算 mean/std，零填充会稀释统计量（std 偏小），归一化后信号幅度会放大。修复：只对信号区域计算统计量，再应用于全数组。

### 8.5 为什么用 Asymmetric Loss 而非 BCE？

27 个类别严重不平衡：窦性心律 20,846 vs 窦性心律失常 43（485:1）。ASL 对负类施加更强聚焦(γ_neg=4)，防止简单负样本主导梯度。

### 8.6 为什么 Agent 用 ReAct 而非 Chain-of-Thought？

ReAct (Reasoning + Acting) 允许 Agent 在推理过程中**实际调用工具获取信息**，而 CoT 只能基于已有知识推理。对于需要精确测量(心率/QT)的医疗场景，工具调用是必需的。

### 8.7 为什么 LLM 接口用 OpenAI 兼容格式？

DeepSeek API、OpenAI API、vLLM 本地部署都支持 `/v1/chat/completions` 端点。统一接口意味着切换 LLM 只需改一行配置。
