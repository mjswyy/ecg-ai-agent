## 🔴 严重（会崩溃或静默产生错误结果）

### 1. `--quick-test` 模式下 `_cosine_schedule` 的 `T_max` 为负数导致崩溃 ✅ 已修复

- **文件**：[trainer.py](../src/ecg_models/trainer.py)：501-502，[train_backbone.py](../scripts/train_backbone.py)：85-86
- **说明**：`--quick-test` 将 epochs 限制为 2，但 `warmup_epochs` 默认是 5。`_cosine_schedule` 中 `T_max = epochs - warmup` = 2 - 5 = -3，PyTorch 的 `CosineAnnealingLR` 要求 `T_max > 0`，会抛 `ValueError`。
- **修复**：`T_max = max(epochs - warmup, 1)`

### 2. `minmax` 归一化静默地将信号数据归零 ✅ 已修复

- **文件**：[preprocessor.py](../src/data_pipeline/preprocessor.py)：309-331
- **说明**：`_normalize` 预分配 `normalized = np.zeros_like(ecg)`，然后仅对 `"zscore"` 分支赋值。若 `normalization` 为 `"minmax"`，循环直接跳过，函数返回全零数组。
- **修复**：补充 `elif self.normalization == "minmax"` 分支，实现 `(x - min) / (max - min)`，含恒定信号保护。

## 🟠 高危（很可能导致训练问题）

### 3. 配置文件从未被训练脚本加载

- **文件**：[train_config.yaml](configs/train_config.yaml)、[model_config.yaml](configs/model_config.yaml)、[train_backbone.py](scripts/train_backbone.py)
- **说明**：两个配置 YAML 有详细文档，但 `train_backbone.py` 和 `train_modelarts.py` 均不读取它们，所有参数用 argparse 默认值。具体不一致：`model_config.yaml` 写 `resnet1d_34.base_channels: 32`，但代码默认值是 `64`。修改 YAML 不会产生任何效果。

### 4. AsymmetricLoss 负类聚焦权重与论文公式不一致

- **文件**：[arrhythmia_classifier.py](src/ecg_models/classifiers/arrhythmia_classifier.py)：103-106
- **说明**：代码注释声称「正确实现了论文概率偏移」。但 ASL 论文（Ben-Baruch et al., ICCV 2021）的负类权重是 `(max(p-m, 0))^gamma_neg`，代码用的是 `(1-p)^gamma_neg`。两个公式策略完全相反：论文给难负样本（模型错误给出高概率）更高权重，代码给难负样本更低权重。当 `gamma_neg=4, p=0.9`（真负类），论文权重 = 0.52（高惩罚），代码权重 = 0.0001（几乎静默）。注释不准确，需澄清。

### 5. ECGAugmentor RNG 在 DataLoader 多进程中产生相同增强序列

- **文件**：[augmentor.py](src/data_pipeline/augmentor.py)：84，[dataset.py](src/data_pipeline/dataset.py)：399，[train_backbone.py](scripts/train_backbone.py)：81
- **说明**：`ECGAugmentor(random_seed=42)` 在 `train_backbone.py` 中只创建一次，通过 pickle 传递给所有 DataLoader worker。所有 worker 的 `np.random.Generator` 初始状态相同。`_worker_init_fn` 只设置 `np.random.seed()`，但 `ECGAugmentor` 用的是独立的 `Generator` 实例，不受影响。结果：同一样本索引在不同 worker 中产生完全相同的增强。Linux fork 模式下状态自然分叉，所以主要在 Windows 上受影响。

### 6. 空标签导致 batch 崩溃

- **文件**：[dataset.py](src/data_pipeline/dataset.py)：146-153
- **说明**：若 manifest 记录缺少 `"labels"` 字段且没有 `"dx_codes"`（或 `label_extractor` 为 None），`labels` 保持为 `np.array([], dtype=np.float32)` shape=(0,)。PyTorch 的默认 collate 无法堆叠不同 shape 的张量，会崩溃：`RuntimeError: stack expects each tensor to be equal size`。预处理数据应始终有标签，但损坏的 manifest 会产生不透明的错误。

### 7. `find_and_fix_data()` 无备份就修改 manifest 文件

- **文件**：[train_modelarts.py](scripts/train_modelarts.py)：104-110
- **说明**：直接读取 manifest JSON、修改路径、写回同一文件。若脚本中途被杀，manifest 会被截断/损坏。未创建备份。共享 OBS 挂载下并发作业也可能相互覆盖。

### 8. 对比学习数据集使用 `rglob` 可能泄漏测试数据

- **文件**：[dataset.py](src/data_pipeline/dataset.py)：191
- **说明**：`ECGContrastiveDataset` 递归枚举 `data_dir` 下**所有** `.npy` 文件，包括训练集、验证集、测试集。因对比学习不使用标签，可以理解，但意味着骨干网络在预训练时已「见过」测试集的信号。后续微调再在测试集上评估，违反评估卫生。

## 🟡 中危（正式训练前建议修复）

### 9. `_challenge_score` 变量名冲突 — `n_pos` 被遮蔽

- **文件**：[trainer.py](src/ecg_models/trainer.py)：456、469
- **修复**：将循环变量从 `n_pos` 改名为 `n_pos_c`。

### 10. InceptionTime 残差相加后缺少 BN/ReLU ✅ 已修复

- **文件**：[inception_time.py](../src/ecg_models/backbone/inception_time.py)：145-148
- **说明**：`x = x + residual` 后无激活或归一化。标准 ResNet 实践在残差相加后有 ReLU。
- **修复**：`x = F.relu(x + residual)`，同时补充 `import torch.nn.functional as F`。

### 11. AMP GradScaler 跳过 NaN batch 后内部状态不确定

- **文件**：[trainer.py](src/ecg_models/trainer.py)：289-318
- **说明**：PyTorch CUDA 的 GradScaler 跳过 step/update 是安全的，但 Ascend NPU 的 `torch.npu.amp.GradScaler` 可能有额外的内部状态计数器。多次跳过可能导致 scale factor 漂移。需独立验证 NPU GradScaler 行为。

### 12. `ECGContrastiveDataset` 加载数据后无验证

- **文件**：[dataset.py](src/data_pipeline/dataset.py)：201-227
- **说明**：对比学习数据集不检查信号是否为合法 ECG（12 导联、非零方差等）。损坏的 .npy 文件会被静默训练。

### 13. 预处理脚本默认路径是 Windows 硬编码路径

- **文件**：[preprocess_data.py](scripts/preprocess_data.py)：44-48
- **说明**：`default="c:/Users/llyun/Desktop/ecg资料/..."`，在任何其他机器上都会失败。

### 14. 对比预训练和微调使用两个独立的 `ECGTrainer` 实例

- **文件**：[train_backbone.py](scripts/train_backbone.py)：109、137
- **说明**：两个 trainer 有各自独立的 GradScaler。对比训练阶段的 NaN 统计信息丢失，但这是可接受的。

### 15. 测试文件硬编码 Windows 路径

- **文件**：[test_loader.py](tests/test_loader.py)：15-17
- **说明**：在 Linux NPU 容器中这些测试会静默 skip。

### 16. `trainer.py` 在模块级别导入 `GradScaler`/`autocast` — 若 `torch_npu` 延迟安装会过时

- **文件**：[trainer.py](src/ecg_models/trainer.py)：31，[device_utils.py](src/utils/device_utils.py)：106-108
- **说明**：当前 `train_modelarts.py` 用子进程执行训练所以不受影响，但若改为进程内执行 AMP 会静默用错后端。

### 17. `model_config.yaml` 模型参数与代码默认值不一致

- **文件**：[model_config.yaml](configs/model_config.yaml) vs [resnet1d.py](src/ecg_models/backbone/resnet1d.py)：153

| 参数 | YAML 配置 | 代码默认 |
