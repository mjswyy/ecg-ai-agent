# ECG AI Agent — 项目进度报告

> **报告日期**：2026-07-15（更新）
> **项目**：面向心电图分析的情境感知与自主交互学习的 AI Agent
> **平台**：华为云 ModelArts + Ascend NPU 910B

---

## 1. 项目总览

### 1.1 项目目标

构建一个 LLM 驱动的 AI Agent，输入 12 导联心电图（ECG）和患者信息，自动生成诊断报告、推理链和标注波形图。

### 1.2 四层架构

```
Phase 4: AI Agent 框架 ← LLM 规划 → 工具调用 → 自反思 → 诊断报告
Phase 3: 情境建模      ← ECG + 文本 + 元数据 → 多模态融合
Phase 2: ECG 基础模型  ← 4 个骨干网络 + 分类器 + 训练器
Phase 1: 数据管道      ← WFDB 加载 → 滤波 → 归一化 → Dataset
```

## 3. 训练实验记录

### 3.1 实验环境

| 项目 | 配置 |
|------|------|
| 平台 | 华为云 ModelArts Notebook |
| 芯片 | Ascend 910B (Snt9B)，单卡 |
| 框架 | PyTorch 2.1.0 + torch_npu 2.1.0.post10 + CANN 8.0.0 |
| 数据 | PhysioNet 2020 Challenge，43,101 条预处理 ECG |

### 3.2 新增训练功能（2026-07-15 更新）

在基础训练管线之上新增了多项防过拟合与泛化增强功能：

| 功能 | 参数 | 作用 | 原理 |
|------|------|------|------|
| 标签平滑 | `--label-smoothing 0.1` | 防模型过自信 | 正样本1→0.9，负样本0→0.1 |
| 梯度噪声 | `--grad-noise 0.001` | 趋向平坦极小值 | backward后加N(0,σ²)噪声 |
| 增强概率控制 | `--augment-prob 0.8` | 增加数据多样性 | 80%概率应用每种增强 |
| 权重衰减 | `--weight-decay 1e-3` | L2正则化 | 抑制权重增长 |
| Dropout | `--dropout 0.3` | 随机丢弃神经元 | 防共适应 |
| Loss选择 | `--loss bce/asl` | BCE更稳定 | ASL与标签平滑可能冲突 |
| 逐类最优阈值 | `evaluate()` 自动搜索 | F1对齐真实能力 | P-R曲线取F1极大点 |
| 早停min_delta | `1e-4` | 微小改善不重置计数器 | 防止训练无效延长 |

### 3.3 InceptionTime 基线 ✅

**训练配置**：lr=1e-4, batch_size=256, epochs=50, AMP 开启

**结果**：

| 指标 | 值 | 说明 |
|------|-----|------|
| val_auc 峰值 | 0.593 | epoch 5 达峰后微跌 |
| macro_auc (test) | ~0.59 | 轻量基线合理水平 |
| macro_f1 (test) | ~0.29 | 逐类最优阈值优化后 |
| 训练时间 | ~40 min | 291K参数快速收敛 |
| 过拟合 | 轻微 | train loss 快速归零，val_auc 峰值后微跌 |

**结论**：291K 参数是硬上限，无论怎么调正则化 val_auc 天花板就是 ~0.60。管线验证通过，作为轻量基线记录。

### 3.4 xResNet1D-101 ✅

**最终训练配置**（经过 3 轮调优确定）：

```
lr=5e-5, batch_size=128, epochs=50
weight_decay=1e-3, dropout=0.3, label_smoothing=0.1
loss=BCE (AsymmetricLoss 与 label_smoothing 冲突导致崩溃)
```

**调优过程**：

| 轮次 | 配置 | 结果 | 结论 |
|:---:|------|------|------|
| 1 | ASL, lr=1e-5, wd=5e-4, drop=0.2 | test_auc=0.544 | 崩溃式过拟合 |
| 2 | BCE, lr=5e-5, wd=1e-3, drop=0.3, ls=0.1 | **test_auc=0.791** | ✅ 大幅改善 |

**关键发现**：
- **lr=1e-5 太低**：导致模型陷入尖锐极小值，过拟合后崩溃
- **AsymmetricLoss + label_smoothing 冲突**：两者组合导致数值不稳定
- **BCE + 强正则化**是大模型从头训练的正确策略

**最终结果**：

| 指标 | 值 | 说明 |
|------|-----|------|
| 训练时间 | ~83 min | 7.6M参数，NPU单卡 |
| 早停 epoch | 46 | val_auc 在 epoch 36 达峰值后 10 轮无改善 |
| val_auc (best) | **0.793** | epoch 36 |
| macro_auc (test) | **0.791** | 与 val_auc 几乎无差，泛化良好 |
| macro_f1 (test) | 0.291 | 逐类最优阈值 |
| challenge_score (test) | 0.390 | 官方竞赛分 |
| mAP (test) | 0.249 | 稀有类严重拖累 |
| 无 NaN | ✅ | 全程稳定 |

**指标解读**：
- `macro_auc=0.791` 是模型真实判别力的最可信指标（不依赖阈值，不惧不平衡）
- `macro_f1=0.291` 低是因为27类极端不平衡（485:1），稀有类（如窦性心律失常仅43样本）的F1极低
- `challenge_score=0.390` 包含排序质量G_beta，稀有类难以做好排序
- `mAP=0.249` 同理，6,470个样本中找43个正样本，少量假阳性排前面就狂跌

**与文献对比**：

| 方案 | macro_auc | 备注 |
|------|:---:|------|
| 随机基线 | 0.50 | — |
| InceptionTime（我们） | 0.59 | 轻量基线 |
| **xResNet1D-101（我们）** | **0.791** | 单模型、无预训练 |
| Challenge Top 10 单模型 | 0.80-0.83 | 文献报告 |
| Challenge 冠军集成 | 0.85+ | 多模型集成 |

### 3.5 ECG Transformer ⏳

**训练配置**：lr=5e-5, batch_size=64, epochs=50, weight_decay=1e-3, dropout=0.3, label_smoothing=0.1

**预期**：单模型 macro_auc 0.76-0.78，与 xResNet 集成后目标 0.83+。

### 3.6 学习率经验总结（更新）

| 骨干 | 参数量 | 推荐 lr | 关键配置 |
|------|--------|---------|----------|
| InceptionTime | 291K | 1e-4 | loss=asl 可，正则化帮助有限 |
| xResNet1D-101 | 7.6M | **5e-5** | loss=bce, wd=1e-3, drop=0.3, ls=0.1 |
| ECG Transformer | 6.5M | 5e-5 | 同上 |

> 经验修正：大模型 lr 不应一味降低——1e-5 导致尖锐极小值过拟合，5e-5 配合强正则化效果更好。

## 4. 关键技术决策

| # | 决策 | 理由 |
|----|------|------|
| 1 | SOS 滤波替代 ba 形式 | 4 阶 Butterworth 的 ba 系数在窄带时数值不稳定 |
| 2 | `resample_poly` 替代线性插值 | 保留 QRS 波群高频形态 |
| 3 | 填充感知归一化 | 零填充区域会稀释 std，导致信号幅度放大 ~2x |
| 4 | BCE + 强正则化替代 ASL | ASL 与 label_smoothing 冲突导致大模型崩溃 |
| 5 | ReAct 模式替代纯 CoT | Agent 需要实际调用工具获取精确测量值 |
| 6 | OpenAI 兼容 LLM 接口 | 一套接口支持 DeepSeek/OpenAI/vLLM |
| 7 | 早停指标从 val_f1 改为 val_auc | val_f1 在极端不平衡下卡在随机基线 |
| 8 | Notebook 直接训练替代训练作业 | 训练作业调试效率极低（每轮 10 分钟） |
| 9 | 标签平滑(0.1)+梯度噪声(0.001) | 防模型过自信，趋向平坦极小值 |
| 10 | 大模型 lr=5e-5 而非 1e-5 | 避免尖锐极小值，跳出坏区域 |
