# ECG AI Agent — 项目进度报告

> **报告日期**：2026-07-15
> **项目**：面向心电图分析的情境感知与自主交互学习的 AI Agent
> **平台**：华为云 ModelArts + Ascend NPU 910B

---

## 1. 项目总览

### 1.1 项目目标

构建一个 LLM 驱动的 AI Agent，输入 12 导联心电图（ECG）和患者信息，自动生成诊断报告、推理链和标注波形图。

### 1.2 四层架构

```
Phase 4: AI Agent 框架 ← LLM 规划 → 工具调用 → 自反思 → 诊断报告
Phase 3: 情境建模      ← ECG + 文本 + 元数据 → 多模态融合
Phase 2: ECG 基础模型  ← 4 个骨干网络 + 分类器 + 训练器
Phase 1: 数据管道      ← WFDB 加载 → 滤波 → 归一化 → Dataset
```

## 4. 关键技术决策

| # | 决策 | 理由 |
|----|------|------|
| 1 | SOS 滤波替代 ba 形式 | 4 阶 Butterworth 的 ba 系数在窄带时数值不稳定 |
| 2 | `resample_poly` 替代线性插值 | 保留 QRS 波群高频形态 |
| 3 | 填充感知归一化 | 零填充区域会稀释 std，导致信号幅度放大 ~2x |
| 4 | Asymmetric Loss 替代 BCE | 处理 485:1 极端类别不平衡 |
| 5 | ReAct 模式替代纯 CoT | Agent 需要实际调用工具获取精确测量值 |
| 6 | OpenAI 兼容 LLM 接口 | 一套接口支持 DeepSeek/OpenAI/vLLM |
| 7 | 早停指标从 val_f1 改为 val_auc | val_f1 在极端不平衡下卡在随机基线 |
| 8 | Notebook 直接训练替代训练作业 | 训练作业调试效率极低（每轮 10 分钟） |
