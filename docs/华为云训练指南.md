### 3.5 验证数据完整性

OBS 上传和同步可能遗漏个别文件。运行完整性检查：

```bash
python -c "
import os, json

processed = '/cache/data/processed'
missing = 0
for split in ['train', 'val', 'test']:
    with open(f'{processed}/{split}_manifest.json') as f:
        data = json.load(f)
    for record in data['files']:
        path = os.path.join(processed, record['signal_file'])
        if not os.path.exists(path):
            missing += 1
            if missing <= 10:
                print(f'缺失: {record[\"signal_file\"]}')
print(f'总计缺失: {missing} 个文件')
"
```

如果缺失文件 ≤ 20 个（<0.05%），直接从 manifest 剔除即可继续训练：

```bash
python -c "
import os, json

processed = '/cache/data/processed'
for split in ['train', 'val', 'test']:
    path = f'{processed}/{split}_manifest.json'
    with open(path) as f:
        data = json.load(f)
    old = len(data['files'])
    data['files'] = [r for r in data['files']
                     if os.path.exists(os.path.join(processed, r['signal_file']))]
    data['count'] = len(data['files'])
    with open(path, 'w') as f:
        json.dump(data, f, indent=2)
    print(f'{split}: {old} → {data[\"count\"]}')
"
```

## 4. 提交正式训练作业

### 4.1 训练顺序

| 序号 | 骨干 | 参数量 | Batch | Epochs | 预计时间 |
|------|------|--------|-------|--------|----------|
| 1 | InceptionTime | ~252K | 256 | 50 | ~2h |
| 2 | xResNet1D-101 | ~7.6M | 128 | 50 | ~6h |
| 3 | ECG Transformer | ~6.5M | 64 | 50 | ~8h |

### 4.2 提交方式：ModelArts 控制台

1. ModelArts 控制台 → **训练管理** → **训练作业** → **创建**
2. 配置：
   - **名称**: `ecg-inception`
   - **算法来源**: 自定义
   - **代码目录**: `obs://ecg-ai-agent/code/ecg-ai-agent/`
   - **启动文件**: `scripts/train_modelarts.py`
   - **AI引擎**: PyTorch 2.1.0 + Ascend
   - **资源池**: Ascend 910B (Snt9B)，单节点 1 卡
   - **环境变量**:
     - `BACKBONE=inception_time`
     - `EPOCHS=50`
     - `BATCH_SIZE=256`
   - **数据输入**: `obs://ecg-ai-agent/data/physionet2020/processed/`
   - **输出路径**: `obs://ecg-ai-agent/outputs/inception_time/`
   - **日志路径**: `obs://ecg-ai-agent/logs/inception_time/`

### 4.3 批量提交

```bash
BACKBONES=("inception_time" "xresnet1d_101" "ecg_transformer")
BATCHES=("256" "128" "64")

for i in "${!BACKBONES[@]}"; do
    ma job create \
        --name "ecg-${BACKBONES[$i]}" \
        --code-dir obs://ecg-ai-agent/code/ecg-ai-agent/ \
        --boot-file scripts/train_modelarts.py \
        --framework PyTorch --framework-version 2.1.0 --engine Ascend \
        --flavor ascend.8vcpu.64g.npu910b --node-count 1 \
        --parameter BACKBONE=${BACKBONES[$i]} \
        --parameter BATCH_SIZE=${BATCHES[$i]} \
        --parameter EPOCHS=50 \
        --input data=obs://ecg-ai-agent/data/physionet2020/processed/ \
        --output checkpoints=obs://ecg-ai-agent/outputs/${BACKBONES[$i]}/ \
        --log obs://ecg-ai-agent/logs/${BACKBONES[$i]}/
done
```

## 6. 常见问题排查

### 6.1 NPU 不可用

```bash
npu-smi info
echo $ASCEND_HOME
python -c 'import torch_npu; print(torch.npu.is_available())'
```

**原因**: CANN / torch_npu / PyTorch 版本不匹配。使用预置镜像 `2.1.0-cann8.0.0` 确保对齐。

### 6.2 训练 NaN

NPU 上数值精度与 CUDA 有差异，训练 NaN 按以下顺序排查：

**第一层：确认基础管道能跑**
```bash
# 小 batch + 低 lr + BCE Loss（默认已启用）
python scripts/train_backbone.py \
    --backbone inception_time \
    --data-dir /cache/data/processed \
    --epochs 2 --quick-test \
    --device auto --batch-size 64 --lr 5e-5
```

**第二层：尝试 FP32（关闭 AMP）**
```bash
# 如果仍有 NaN，加 --no-amp
python scripts/train_backbone.py ... --no-amp
```

**第三层：进一步降低 lr**
```bash
--lr 1e-5 --batch-size 32
```

**已内置的防护**：
- 梯度 NaN/Inf 检测：backward 后检查所有参数的 grad，含 NaN 则跳过该 step
- Loss NaN 检测：forward 后检查 loss，含 NaN 则跳过该 batch
- AsymmetricLoss logit 钳制：logits 截断到 [-50, 50] 防止 sigmoid 溢出
- 默认 loss 改为 `BCEWithLogitsLoss`（数值比 AsymmetricLoss 更稳定）

### 6.3 NumPy 版本冲突

```
A module that was compiled using NumPy 1.x cannot be run in NumPy 2.2.6
```

**修复**：
```bash
pip install "numpy<2"
```

### 6.4 内存不足

Ascend 910B 有 ~61 GB HBM，对于本项目模型绰绰有余。如果 OOM：

```bash
export PYTORCH_NPU_ALLOC_CONF=expandable_segments:True
python scripts/train_backbone.py ... --batch-size 32
```

### 6.5 DataLoader 慢

```bash
# num_workers 用 0 可排除多进程问题，调试通过后恢复 4
--num-workers 4
```

### 6.6 OBS 同步无数据

`moxing` 同步后数据在 `/cache/data/processed/`，「不是在 `/home/ma-user/work/`，JupyterLab 文件浏览器看不到 `/cache/`。终端 `ls /cache/data/processed/` 验证。

### 6.7 git pull "Already up to date" 但代码没变

说明本地改动未推送到 GitHub。先确认：

```powershell
# 本地 Windows
git log --oneline -3   # 确认最新 commit
git push origin main   # 推送到 GitHub
```

Notebook 上：
```bash
git fetch origin && git reset --hard origin/main
```

### 6.8 Bash 语法错误 `syntax error near unexpected token`

终端中 `python -c "..."` 双引号内的 `$` 和括号会被 bash 解析。用**单引号**包裹 Python 代码：

```bash
python -c '
import torch
print(torch.__version__)
'
```
