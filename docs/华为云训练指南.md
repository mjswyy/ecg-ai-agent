## 4. 提交正式训练作业

### 4.1 提交方式一：ModelArts 控制台

1. ModelArts 控制台 → **训练管理** → **训练作业** → **创建**
2. 配置参数:
   - **名称**: `ecg-backbone-inception`
   - **算法来源**: 自定义
   - **代码目录**: `obs://ecg-ai-agent/code/ecg-ai-agent/`
   - **启动文件**: `scripts/train_modelarts.py`
   - **AI引擎**: PyTorch 2.1.0 + Ascend
   - **资源池**: Ascend 910B (Snt9B)
   - **节点数**: 1
   - **环境变量**:
     - `BACKBONE=inception_time`
     - `EPOCHS=50`
     - `BATCH_SIZE=128`
   - **数据输入**: `obs://ecg-ai-agent/data/physionet2020/processed/`
   - **输出路径**: `obs://ecg-ai-agent/outputs/inception_time/`
   - **日志路径**: `obs://ecg-ai-agent/logs/inception_time/`

### 4.2 提交方式二：ModelArts CLI (ma-cli)

```bash
# 安装 ma-cli
pip install modelarts-cli

# 提交训练作业
ma job create \
    --name ecg-backbone-inception \
    --code-dir obs://ecg-ai-agent/code/ecg-ai-agent/ \
    --boot-file scripts/train_modelarts.py \
    --framework PyTorch \
    --framework-version 2.1.0 \
    --engine Ascend \
    --flavor ascend.8vcpu.64g.npu910b \
    --node-count 1 \
    --parameter BACKBONE=inception_time \
    --parameter EPOCHS=50 \
    --parameter BATCH_SIZE=128 \
    --input data=obs://ecg-ai-agent/data/physionet2020/processed/ \
    --output checkpoints=obs://ecg-ai-agent/outputs/inception_time/ \
    --log obs://ecg-ai-agent/logs/inception_time/
```

### 4.3 训练顺序和预计时间

按优先级顺序提交，建议先跑 InceptionTime 验证环境，再跑其他:

| 序号 | 骨干 | 参数量 | Batch | Epochs | 预计时间 | 备注 |
|------|------|--------|-------|--------|----------|------|
| 1 | InceptionTime | ~252K | 256 | 50 | ~2h | 基线，先跑通验证环境 |
| 2 | xResNet1D-101 | ~7.6M | 128 | 50 | ~6h | 挑战赛冠军方案 |
| 3 | ECG Transformer | ~6.5M | 64 | 50 | ~8h | 创新骨干 |

### 4.4 批量提交脚本

```bash
#!/bin/bash
# 提交所有三个骨干的训练作业

BACKBONES=("inception_time" "xresnet1d_101" "ecg_transformer")
BATCHES=("256" "128" "64")

for i in "${!BACKBONES[@]}"; do
    backbone=${BACKBONES[$i]}
    batch=${BATCHES[$i]}

    ma job create \
        --name "ecg-${backbone}" \
        --code-dir obs://ecg-ai-agent/code/ecg-ai-agent/ \
        --boot-file scripts/train_modelarts.py \
        --framework PyTorch --framework-version 2.1.0 --engine Ascend \
        --flavor ascend.8vcpu.64g.npu910b --node-count 1 \
        --parameter BACKBONE=${backbone} \
        --parameter BATCH_SIZE=${batch} \
        --parameter EPOCHS=50 \
        --input data=obs://ecg-ai-agent/data/physionet2020/processed/ \
        --output checkpoints=obs://ecg-ai-agent/outputs/${backbone}/ \
        --log obs://ecg-ai-agent/logs/${backbone}/

    echo "Submitted: ${backbone}"
done
```

## 6. 常见问题排查

### 6.1 NPU 不可用 (`torch.npu.is_available()` 返回 False)

```bash
# 检查 CANN 驱动
npu-smi info

# 检查环境变量
echo $ASCEND_HOME
echo $ASCEND_RT_VISIBLE_DEVICES

# 确认版本匹配
python -c "import torch_npu; print(torch_npu.__version__)"
```

**根本原因**: CANN / torch_npu / PyTorch 版本不匹配。请使用 ModelArts 预置镜像确保版本对齐。

### 6.2 AMP 训练 NaN

在 Ascend NPU 上 AMP 的 loss scaling 行为与 CUDA 不同，训练可能出现 NaN。代码中已配置 `grad_clip=1.0` 梯度裁剪。如果仍然出现 NaN:

```bash
# 解决方案1: 减小学习率
--lr 5e-5

# 解决方案2: 增大 gradient clipping
# 修改 train_config.yaml: grad_clip: 0.5

# 解决方案3: 先用 FP32 跑通，再切 AMP
--no-amp
```

### 6.3 内存不足 (OOM)

Ascend 910B 有 ~61 GB HBM，对于本项目模型绰绰有余。如果出现 OOM:

```bash
# 启用内存碎片优化
export PYTORCH_NPU_ALLOC_CONF=expandable_segments:True

# 减小 batch size
--batch-size 64  # 改为更小

# 清理缓存
# 代码中已自动调用 torch.npu.empty_cache() (在 GradScaler.update() 中处理)
```

### 6.4 DataLoader 速度慢

```bash
# 增加数据加载 workers
--num-workers 8

# 确保数据在本地缓存 /cache/data（非 OBS 直接读取）
# 代码中 train_modelarts.py 已自动执行 moxing 同步
```

### 6.5 OBS 数据同步失败

```bash
# 手动同步（在 Notebook 中）
python -c "
import moxing as mox
mox.file.copy_parallel('obs://ecg-ai-agent/data/', '/cache/data/')
"

# 验证同步
ls -la /cache/data/
```
