```bash
python -c '
import os, json

processed = "/cache/data/processed"
missing = 0
for split in ["train", "val", "test"]:
    with open(f"{processed}/{split}_manifest.json") as f:
        data = json.load(f)
    for record in data["files"]:
        path = os.path.join(processed, record["signal_file"])
        if not os.path.exists(path):
            missing += 1
            if missing <= 10:
                sig = record["signal_file"]
                print(f"缺失: {sig}")
print(f"总计缺失: {missing} 个文件")
'
```

如果缺失文件 ≤ 20 个（<0.05%），直接从 manifest 剔除即可继续训练：

```bash
python -c '
import os, json

processed = "/cache/data/processed"
for split in ["train", "val", "test"]:
    path = f"{processed}/{split}_manifest.json"
    with open(path) as f:
        data = json.load(f)
    old = len(data["files"])
    data["files"] = [r for r in data["files"]
                     if os.path.exists(os.path.join(processed, r["signal_file"]))]
    new_count = len(data["files"])
    data["count"] = new_count
    with open(path, "w") as f:
        json.dump(data, f, indent=2)
    print(f"{split}: {old} → {new_count}")
'
```

## 4. 正式训练

> **2026-07-15 更新**：经过三轮训练作业调试（路径发现、依赖安装、数据挂载），发现训练作业调试效率极低（每次改代码 → OBS 上传 → 提交作业 → 等日志 → 失败 → 循环 10 分钟）。最终决定**直接在 Notebook 终端上训练**，训练作业作为批量提交备用方案。

### 4.1 Notebook 直接训练（推荐）

#### 4.1.1 训练顺序与参数

| 序号 | 骨干 | 参数量 | Batch | lr | Epochs | 预计时间 | 状态 |
|------|------|--------|-------|-----|--------|----------|:---:|
| 1 | InceptionTime | ~291K | 256 | 1e-4 | 50 | ~40min | ✅ 完成 (test_auc=0.59) |
| 2 | xResNet1D-101 | ~7.6M | 128 | 5e-5 | 50 | ~83min | ✅ 完成 (test_auc=0.791) |
| 3 | ECG Transformer | ~6.5M | 64 | 5e-5 | 50 | ~8h | ⏳ 待训练 |

> **lr 选择依据（2026-07-15 更新）**：InceptionTime (291K) 用 1e-4。大模型 **lr 不应一味降低**——1e-5 导致 xResNet 陷入尖锐极小值过拟合（test_auc=0.544），5e-5 配合强正则化效果更好（test_auc=0.791）。

#### 4.1.2 训练命令（含全部防过拟合参数）

```bash
# 1. InceptionTime 基线（~40 分钟）
# 291K 参数轻量模型，快速验证管线
python scripts/train_backbone.py \
    --backbone inception_time \
    --data-dir /cache/data/processed \
    --epochs 50 \
    --device auto \
    --batch-size 256 \
    --lr 1e-4

# 2. xResNet1D-101（~83 分钟）✅ 推荐配置
# 关键: lr=5e-5 (非1e-5!), loss=bce, 强正则化
python scripts/train_backbone.py \
    --backbone xresnet1d_101 \
    --data-dir /cache/data/processed \
    --epochs 50 \
    --device auto \
    --batch-size 128 \
    --lr 5e-5 \
    --weight-decay 1e-3 \
    --dropout 0.3 \
    --label-smoothing 0.1 \
    --loss bce

# 3. ECG Transformer（~8 小时）
python scripts/train_backbone.py \
    --backbone ecg_transformer \
    --data-dir /cache/data/processed \
    --epochs 50 \
    --device auto \
    --batch-size 64 \
    --lr 5e-5 \
    --weight-decay 1e-3 \
    --dropout 0.3 \
    --label-smoothing 0.1 \
    --loss bce
```

> **全部 CLI 参数**：`--weight-decay`（L2正则化）、`--dropout`（dropout率）、`--label-smoothing`（标签软化）、`--augment-prob`（增强概率，默认1.0）、`--grad-noise`（梯度噪声，默认0）、`--loss`（bce/asl）、`--patience`（早停耐心）。
>
> **注意**：Notebook 训练时输出目录默认为 `/cache/output/`。训练完成后用 moxing 同步权重到 OBS 防止丢失：
> ```bash
> python -c 'import moxing as mox; mox.file.copy_parallel("/cache/output/", "obs://ecg-ai-agent/outputs/模型名/")'
> ```

### 6.2 训练 NaN / 评估全 0

NPU 上训练 NaN 按以下顺序排查，由浅入深：

**第零层：检查输入数据是否含 NaN**

大部分 NPU NaN 问题的真正根因。运行输入数据扫描：

```bash
python -c '
import os, json, numpy as np

processed = "/cache/data/processed"
nan_files = []
total = 0
for split in ["train", "val", "test"]:
    with open(f"{processed}/{split}_manifest.json") as f:
        data = json.load(f)
    total += len(data["files"])
    for r in data["files"]:
        sig = np.load(os.path.join(processed, r["signal_file"]))
        if np.isnan(sig).any() or np.isinf(sig).any():
            nan_files.append(r["signal_file"])
            if len(nan_files) <= 5:
                n = np.isnan(sig).sum() + np.isinf(sig).sum()
                name = r["signal_file"]
                print(f"  {name}: {int(n)} NaN/Inf")
print(f"总计: {len(nan_files)} / {total} 文件含 NaN")
'
```

如果发现 NaN 文件（本项目实测 8/43,101），确认 Dataset 代码已包含 `np.nan_to_num()` 清洗（详见 [踩坑记录 1.6](踩坑记录.md#16-预处理数据含-nan-导致训练全量静默失败)）。

**第一层：尝试关闭 AMP**

```bash
# 加 --no-amp 使用纯 FP32
python scripts/train_backbone.py \
    --backbone inception_time \
    --data-dir /cache/data/processed \
    --epochs 2 --quick-test \
    --device auto --batch-size 64 --lr 5e-5 \
    --no-amp
```

> AMP 在 NPU 上经实测正常可用（InceptionTime 2 epoch 无 NaN），关闭 AMP 仅作为排查手段。

**第二层：进一步降低学习率**

```bash
--lr 1e-5 --batch-size 32
```

**已内置的防护**：
- 梯度 NaN/Inf 检测：backward 后检查所有参数的 grad，含 NaN 则跳过该 step
- Loss NaN 检测：forward 后检查 loss，含 NaN 则跳过该 batch
- AsymmetricLoss logit 钳制：logits 截断到 [-50, 50] 防止 sigmoid 溢出
- 默认 loss 为 `BCEWithLogitsLoss`（数值比 AsymmetricLoss 更稳定）
- Dataset 层 `np.nan_to_num()` 清洗输入数据的 NaN/Inf

> **注意**：以上 NaN 保护机制有根本缺陷 — `backward()` 之后才检测 NaN，梯度可能已经污染模型权重。冒烟测试中若 val_auc = 0.5000（随机基线）且 logits 100% NaN，基本是数据问题而非模型问题。详见 [踩坑记录 1.6](踩坑记录.md#16-预处理数据含-nan-导致训练全量静默失败)。
